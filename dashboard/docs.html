WishlistOps

Steam announcements from Git commits

WishlistOps turns player-facing commits into an announcement draft, makes a Steam-ready banner from your screenshots, and posts everything to Discord for human approval.

‚úÖ
No AI image generation. WishlistOps only processes user-provided screenshots/logos (crop/resize/overlay). It does not invent new images.

What it does

WishlistOps is an AI co-pilot for Steam marketing. It:

  • Reads feat:/fix: commits (player-facing) and ignores internal noise.
  • Generates a concise announcement draft using your selected Gemini model.
  • Runs an anti-slop quality filter (avoids corporate buzzwords).
  • Builds an 800√ó450 banner from a screenshot and optionally overlays your logo.
  • Sends everything to Discord via webhook for approval (human-in-the-loop).

Workflow

The default workflow is:

  • Choose a repo/project (Settings ‚Üí Projects)
  • Load commits (Generate)
  • Pick commits + pick a screenshot (or let it auto-select)
  • Generate draft + preview banner
  • Send to Discord for approval

Tip: Add a screenshot directive in a commit message like [shot: screenshots/my_banner.png] to deterministically pick an image.

Images

WishlistOps expects you to provide:

  • A logo (recommended: transparent PNG)
  • Screenshots placed under screenshots/ (or uploaded via the dashboard)

You can control cropping and logo overlay from the Generate view.

Multiple repos & games

WishlistOps can manage multiple local Git repos (‚Äúprojects‚Äù) so you can generate announcements for different games without constantly editing files.

  • Each project has a repo path and a config path.
  • Switching the active project updates which commits/images/config are used for generation.
  • Secrets remain local to each repo via its own .env.

CLI commands

From a terminal in your repo:

  • python -m wishlistops.main run --dry-run --verbose
  • python -m wishlistops.main setup --port 8080

If you prefer dashboard-only, open the dashboard and use Generate.
