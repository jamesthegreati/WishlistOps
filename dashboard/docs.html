üöÄ Getting Started

What is WishlistOps?

WishlistOps automatically transforms your Git commits into Steam announcements, saving you hours of marketing work every week. It monitors your game repository, generates announcements using AI, and sends them to Discord for your approval before posting to Steam.

How It Works

  1. You code: Work on your game normally, making commits as usual
  2. You tag: When ready for an announcement, create a Git tag (e.g., v1.2.0)
  3. AI generates: WishlistOps reads your commits and generates a Steam announcement
  4. You approve: Review the draft in Discord and approve it
  5. Auto-posts: WishlistOps posts to Steam and notifies you
üí° Pro Tip: Set up WishlistOps once and forget about it. It runs automatically every time you tag a release!

Quick Start (2 Minutes)

  1. Run pip install wishlistops
  2. Run wishlistops setup - this opens the setup wizard
  3. Connect GitHub, Discord, and Google AI (guided with screenshots)
  4. Select your game repository
  5. Tag your next release: git tag v1.0.0 && git push origin v1.0.0
  6. Check Discord for your announcement draft!

üìù Commit Message Conventions

WishlistOps works best when you use clear, descriptive commit messages. The AI reads your commits to understand what changed and writes announcements accordingly.

Commit Types (Conventional Commits)

Start your commits with a type prefix to help WishlistOps categorize changes:

‚úÖ GOOD: feat: added boss fight in Forest Temple
Clear feature addition that players care about
‚úÖ GOOD: fix: enemies no longer spawn inside walls
Bug fix that improves gameplay
‚úÖ GOOD: content: new medieval armor set with 15 variants
New content players will appreciate
‚ùå BAD: update stuff
Too vague, AI can't write a meaningful announcement
‚ùå BAD: refactor: reorganized inventory code
Internal change, not player-facing

Recommended Prefixes

feat:     New gameplay features (bosses, levels, mechanics)
content:  New content (items, characters, quests)
fix:      Bug fixes that improve player experience
balance:  Gameplay balance changes
improve:  Improvements to existing features
audio:    Sound effects or music updates
visual:   Graphics, animations, or visual effects

Writing Player-Facing Commits

üéØ Golden Rule: Write commits as if you're talking to your players, not your code.
Instead of... Write...
fixed null reference fix: game no longer crashes when opening inventory
added new class feat: new Necromancer class with skeleton summoning
performance improvements improve: game now runs 30% faster on low-end PCs

Using Screenshot Tags

Link screenshots to your commits for better banners:

git commit -m "feat: new underwater level [shot: screenshots/underwater.png]"

‚≠ê Best Practices

Announcement Frequency

  • Weekly updates: Ideal for active development
  • Bi-weekly: Good for smaller teams
  • Monthly: Minimum to maintain wishlist momentum
üí° Sweet Spot: Post every 7-14 days. More frequent = higher engagement, but avoid daily spam.

What Makes a Great Announcement?

  1. Visual: Include screenshots or GIFs from your game
  2. Specific: "Added 5 new enemies" beats "Various improvements"
  3. Exciting: Highlight the coolest changes first
  4. Honest: Mention bug fixes alongside new features
  5. Community-focused: Thank players for feedback if applicable

Screenshot Guidelines

  • Resolution: Minimum 1920x1080 (1080p)
  • Format: PNG preferred, JPEG acceptable
  • Content: Show exciting gameplay moments, not menus
  • UI: Hide debug overlays and FPS counters
  • Branding: WishlistOps auto-adds your logo

Commit Batching Strategy

Group related commits for better announcements:

# Bad: Announcing every tiny commit
git tag v1.0.1  # "Fixed typo"
git tag v1.0.2  # "Fixed another typo"

# Good: Batch related changes
# Work on multiple features over a week
git tag v1.1.0  # Announces all changes together

Config File Location

WishlistOps stores configuration in wishlistops/config.json

Key Settings

Steam Settings

"steam": {
  "app_id": "480",           // Your Steam App ID
  "app_name": "My Game"      // Display name
}

Branding

"branding": {
  "art_style": "pixel art fantasy",  // For AI image generation
  "logo_path": "assets/logo.png",    // Your game logo
  "logo_position": "top-right",      // Where to place logo
  "logo_size_percent": 25            // Logo size (10-50%)
}

Voice Settings

"voice": {
  "tone": "casual and excited",
  "personality": "friendly indie developer",
  "avoid_phrases": [
    "monetization",
    "grind",
    "pay-to-win"
  ]
}

Automation

"automation": {
  "enabled": true,
  "trigger_on_tags": true,           // Run on git tags
  "min_days_between_posts": 7,       // Rate limiting
  "require_approval": true           // Always true for safety
}

üîß Troubleshooting

No Announcements Generated

Possible causes:

  • No player-facing commits found (use feat:, fix:, etc.)
  • Rate limited (posted too recently, check min_days_between_posts)
  • Not enough commits (minimum 3 by default)

Solution: Check wishlistops/logs/ for details

Discord Not Receiving Messages

Check:

  1. Webhook URL is correct
  2. Channel permissions allow webhooks
  3. Discord server is not muted

Test webhook:

curl -X POST "YOUR_WEBHOOK_URL" \
  -H "Content-Type: application/json" \
  -d '{"content": "Test message"}'

AI Generating Poor Quality

Improve quality by:

  • Writing better commit messages (see Commit Conventions)
  • Adjusting temperature setting (0.3-0.7 for consistency)
  • Adding more context to art_style and personality
  • Using the anti-slop filter (automatically enabled)

Screenshots Not Showing

Verify:

  • Screenshot path in commit message is correct
  • File exists in repository
  • Git LFS is enabled if files are large
  • File format is PNG or JPEG

GitHub Actions Failing

Common issues:

  • Secrets not configured (check Settings ‚Üí Secrets)
  • Workflow file has syntax errors
  • Python dependencies not installing

Debug: Check Actions tab in GitHub for detailed logs

‚ùì Frequently Asked Questions

Is my data private?

Yes! WishlistOps runs entirely on your local machine or in your GitHub Actions. Your API keys and configurations never leave your control. The only external calls are to:

  • Google AI API (to generate text)
  • Discord API (to send notifications)
  • Steam API (to post announcements)

What does it cost?

WishlistOps is free! You only pay for API usage:

  • Google AI: Free tier = 1,500 requests/day (100+ announcements/month)
  • GitHub Actions: Free tier = 3,000 minutes/month (more than enough)
  • Discord: Free
  • Steam: Free (you need a Steamworks partner account)

Can I use it for multiple games?

Yes! WishlistOps supports unlimited projects. Just configure each repository separately in the dashboard.

What if I don't like an announcement?

Every announcement requires your approval in Discord before posting. You can:

  • Approve as-is (‚úÖ)
  • Cancel (‚ùå)
  • Edit the text manually before approving

Does it work with Godot/Unity/Unreal?

Yes! WishlistOps is engine-agnostic. It reads Git commits, not game files. Works with any engine or framework.

Can I customize the announcement style?

Absolutely! Configure the voice settings:

  • tone: "professional", "casual", "excited", etc.
  • personality: "friendly developer", "mysterious storyteller", etc.
  • avoid_phrases: Words you never want to use

What happens if Steam is down?

WishlistOps automatically retries failed posts with exponential backoff. If Steam is down for extended periods, it will notify you in Discord and queue the announcement for later.

How do I uninstall?

Run pip uninstall wishlistops and delete the wishlistops/ folder from your project. Your Git repository remains unchanged.
